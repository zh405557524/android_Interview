# 1.2 Java 集合

## 本章目标

- 建立集合框架的整体地图，理解常见集合的底层结构与选型原则。
- 面试时能从时间复杂度、空间占用、线程安全和使用场景四个角度回答。

## 核心知识点清单

- `List` / `Set` / `Map` 整体体系
- `ArrayList`
- `LinkedList`
- `HashMap`
- `ConcurrentHashMap`
- `HashSet` / `TreeSet`
- `Queue` / `Deque`
- `Collections` 工具类

## 高频面试题

- `ArrayList` 和 `LinkedList` 的底层实现及适用场景是什么？
- `HashMap` 在 JDK 1.8 中做了哪些优化？
- `ConcurrentHashMap` 为什么能做到更高并发？

## 追问方向

- 扩容、负载因子、哈希冲突与红黑树化过程。
- `HashSet`、`TreeSet`、`LinkedHashSet` 的区别。
- 迭代器 fail-fast 机制与并发修改异常。

## 易错点 / 对比点

- `LinkedList` 并不等于“任何插入删除都快”，定位节点本身也有成本。
- `HashMap` 线程不安全，不能直接拿来做高并发共享容器。
- `Collections.synchronizedXxx` 和并发容器不是同一个思路。
