# 4.6 数据库框架

## 本章目标

- 建立 Android 常见数据库与 ORM 框架的对比认知。
- 能从开发效率、性能和可维护性做选型说明。

## 核心知识点清单

- SQLite
- Room
- GreenDao
- LiteOrm
- ORM 思想与优缺点

## 高频面试题

- ORM 框架解决了什么问题？
- Room 相比 GreenDao、LiteOrm 的优势是什么？
- 什么时候直接使用 SQLite 反而更合适？

## 追问方向

- SQL 能力、编译期校验和迁移支持。
- 对象映射开销与复杂查询能力。
- 响应式数据库访问与线程模型。

## 易错点 / 对比点

- ORM 提升的是开发效率，不一定绝对提升性能。
- Room 依然基于 SQLite。
- 复杂场景下仍要关注 SQL 本身和索引设计。
