# 2.2 Activity

## 本章目标

- 建立 Activity 作为页面容器的生命周期和任务栈认知。
- 能把页面恢复、启动模式和配置变更讲清楚。

## 核心知识点清单

- 生命周期
- 启动模式
- task 与 back stack
- `onSaveInstanceState`
- 横竖屏切换与配置变更
- Activity 间数据传递
- Fragment 与 Activity 协同

## 高频面试题

- Activity 生命周期在不同场景下是怎么回调的？
- 四种启动模式分别解决什么问题？
- 横竖屏切换为什么会触发重建，如何保存状态？

## 追问方向

- 冷启动、热启动、返回栈恢复。
- `launchMode` 与 `Intent Flag` 的组合行为。
- `Fragment` 嵌套和 Activity 结果回传演进。

## 易错点 / 对比点

- `onSaveInstanceState` 不等于必然持久化成功。
- 单任务栈不等于全局只有一个 Activity 实例。
- 配置变更自己接管后要考虑资源重载问题。
