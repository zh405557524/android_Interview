# 2.14 Compose 基础

## 本章目标

- 建立 Compose 声明式 UI 的核心认知。
- 能把 Compose 与传统 View 系统的思维差异讲清楚。

## 核心知识点清单

- Compose 基本思想
- 可组合函数
- 状态管理
- 重组机制
- `remember` / `mutableStateOf`
- Compose 与 View 混用
- Compose 性能基础

## 高频面试题

- Compose 为什么是声明式 UI？
- 什么情况下会触发重组，如何避免无效重组？
- Compose 和传统 View 混用时要关注哪些问题？

## 追问方向

- 状态提升、单向数据流。
- `remember`、`rememberSaveable` 和状态持久化。
- 稳定性、跳过重组和性能调优基础。

## 易错点 / 对比点

- 重组不等于整棵界面完全重绘。
- `remember` 只在当前 Composition 生命周期内生效。
- Compose 性能问题很多时候来自状态设计而不是框架本身。
