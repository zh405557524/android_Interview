# 3.3 SystemServer

## 本章目标

- 理解 `SystemServer` 在 Android Framework 中的中心地位。
- 建立系统服务启动、注册和对外暴露的主线认知。

## 核心知识点清单

- `SystemServer` 作用
- 核心系统服务启动
- 服务注册流程
- Java Framework 服务初始化顺序

## 高频面试题

- `SystemServer` 为什么重要？
- 常见系统服务是如何启动并注册到系统中的？
- 服务注册后应用进程如何拿到它们？

## 追问方向

- `SystemServiceManager` 的职责。
- 启动阶段服务的依赖关系与顺序。
- 与 `ServiceManager`、Binder 的关系。

## 易错点 / 对比点

- `SystemServer` 是一个进程，不是单个服务。
- 系统服务不都在 native 层，很多核心服务在 Java Framework 层。
- 注册服务和真正对外提供稳定能力是两层问题。
