# 8.5 项目回答要补的能力点

## 本章目标

- 梳理项目表达中最容易被忽略但很加分的能力维度。
- 把“做过事情”转成“具备能力”的面试表达。

## 核心知识点清单

- 从 0 到 1
- 架构搭建
- 模块化设计
- SDK 设计
- 跨进程通信
- Native 网络
- 语音流程状态机
- 性能优化
- 团队协作与任务拆分

## 高频面试题

- 项目回答里为什么一定要讲能力而不只是任务？
- 如何把具体工作提炼成架构、协作和工程能力？
- 哪些能力点最容易在追问里拉开差距？

## 追问方向

- 独立负责、主导推进与跨团队沟通。
- 方案设计、接口抽象和系统治理。
- 性能优化、稳定性保障和复杂链路排障。

## 易错点 / 对比点

- 不要把“参与开发”包装成“主导设计”。
- 能力点最好绑定具体场景和结果。
- 团队协作能力也需要技术上下文支撑。
