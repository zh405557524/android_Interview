# 3.10 View 绘制原理

## 本章目标

- 从 Framework 角度理解 View 刷新与绘制流程。
- 把 `ViewRootImpl`、`Choreographer`、VSYNC 串成一条主线。

## 核心知识点清单

- `ViewRootImpl`
- `performTraversals`
- `measure` / `layout` / `draw` 流程
- `Choreographer`
- VSYNC
- 刷新机制

## 高频面试题

- View 从请求刷新到真正绘制经历了什么？
- `performTraversals` 为什么重要？
- VSYNC 与 `Choreographer` 如何影响界面刷新节奏？

## 追问方向

- `scheduleTraversals` 和同步屏障。
- `measure/layout/draw` 在 Framework 里的串联方式。
- 掉帧、重绘和 UI 卡顿的关系。

## 易错点 / 对比点

- View 绘制原理和自定义 View 基础要区分层次。
- 并不是每次 `invalidate` 都会立刻同步绘制。
- VSYNC 的核心是节奏驱动，不是单纯“让它更快”。
