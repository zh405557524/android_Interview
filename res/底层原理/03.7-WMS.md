# 3.7 WMS

## 本章目标

- 理解窗口管理从 `Window` 到显示在屏幕上的关键链路。
- 能把 `Window`、`DecorView`、`ViewRootImpl` 的关系讲清楚。

## 核心知识点清单

- Window 管理机制
- `Window` / `DecorView` / `ViewRootImpl`
- `addView` 流程
- 窗口显示流程
- 输入事件与窗口关系

## 高频面试题

- WMS 的职责是什么？
- Activity 界面是如何通过 `Window` 显示出来的？
- `ViewRootImpl` 在 View 系统中扮演什么角色？

## 追问方向

- `WindowManagerGlobal`、`Session`、WMS 的调用链。
- Toast、Dialog、PopupWindow 的窗口差异。
- 窗口层级、token 和权限控制。

## 易错点 / 对比点

- `Window` 不是实际 View，它更像窗口抽象。
- `DecorView` 是顶层 View 容器。
- 输入事件分发和窗口可见性、焦点强相关。
