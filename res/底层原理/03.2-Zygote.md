# 3.2 Zygote

## 本章目标

- 理解 Zygote 作为应用进程孵化器的设计价值。
- 能把 fork 模型、预加载和进程启动收益讲清楚。

## 核心知识点清单

- Zygote 作用
- 孵化应用进程原理
- fork 模型
- 资源预加载

## 高频面试题

- Zygote 为什么要存在？
- 应用进程为什么通过 Zygote fork 出来？
- Zygote 预加载资源带来了什么收益和代价？

## 追问方向

- Socket 通信与进程孵化命令传递。
- Copy-On-Write 带来的内存复用收益。
- `app_process`、Runtime 初始化与 Java 世界入口。

## 易错点 / 对比点

- Zygote 不是普通应用进程。
- 预加载不是越多越好，会影响系统启动时间和内存。
- fork 后子进程还需要完成自己的 Runtime 和应用初始化。
