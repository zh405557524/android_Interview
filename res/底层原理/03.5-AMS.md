# 3.5 AMS

## 本章目标

- 建立 AMS 在组件调度、进程管理和任务栈管理中的核心作用认知。
- 能把 Activity、Service、Broadcast 启动流程串到 AMS 上。

## 核心知识点清单

- AMS 作用
- Activity 启动流程
- Service 启动流程
- Broadcast 分发流程
- 进程管理
- task 管理

## 高频面试题

- AMS 的核心职责有哪些？
- Activity 启动流程中 AMS 主要做了什么？
- AMS 如何参与进程优先级与任务栈管理？

## 追问方向

- 从应用进程发起请求到系统服务调度的链路。
- 进程拉起与组件恢复。
- 任务栈、最近任务和多窗口场景。

## 易错点 / 对比点

- AMS 不是单独负责 Activity，它管理的是应用运行模型。
- Activity 启动流程不能只背应用侧，还要有系统侧视角。
- task 管理和进程管理是相关但不同的维度。
