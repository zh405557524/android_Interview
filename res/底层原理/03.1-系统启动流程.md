# 3.1 系统启动流程

## 本章目标

- 建立 Android 从开机到桌面可交互的整体启动链路认知。
- 为后续 Zygote、SystemServer、AMS 等专题打总纲。

## 核心知识点清单

- 开机启动整体链路
- bootloader
- Linux 内核启动
- `init` 进程
- `init.rc`
- Zygote 启动
- `SystemServer` 启动
- Launcher 启动

## 高频面试题

- Android 系统从按下电源键到桌面显示大致经历了哪些阶段？
- `init`、Zygote、`SystemServer` 分别扮演什么角色？
- 为什么 Zygote 是整个 Android 进程孵化的核心？

## 追问方向

- `init.rc` 的解析和服务拉起机制。
- 启动阶段 native 层和 framework 层的分界点。
- 从系统启动过渡到应用启动的关键节点。

## 易错点 / 对比点

- 系统启动链路不要一开始就陷入源码细节，先讲主干。
- Launcher 启动属于系统就绪后的用户交互入口。
- `SystemServer` 不是第一个用户空间进程。
