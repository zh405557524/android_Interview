# 7.3 语音链路全流程

## 本章目标

- 建立语音交互从唤醒到结果播报的全链路认知。
- 能把多轮对话、打断和状态机思路串起来讲。

## 核心知识点清单

- 唤醒
- VAD
- ASR
- NLU / 语义理解
- TTS
- 对话状态管理
- 子会话 / 多轮对话
- 全双工与半双工
- 语音打断
- 语音状态机设计

## 高频面试题

- 一次完整语音交互链路通常经过哪些阶段？
- VAD、ASR、NLU、TTS 分别负责什么？
- 多轮对话和打断为什么需要状态机设计？

## 追问方向

- 唤醒后链路编排与超时控制。
- 全双工与半双工的交互差异。
- 车机场景下的误唤醒、漏识别和中断恢复。

## 易错点 / 对比点

- 语音链路不是线性顺序执行这么简单，还涉及状态切换。
- NLU 负责理解，不直接等于最终业务执行。
- 打断处理往往是面试里体现系统设计能力的重点。
