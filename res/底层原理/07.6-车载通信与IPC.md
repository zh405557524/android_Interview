# 7.6 车载通信与 IPC

## 本章目标

- 理解车载系统中跨进程服务、SDK 封装和状态同步设计。
- 能结合车机 Launcher、业务 App 和服务层讲通信架构。

## 核心知识点清单

- AIDL 在车机中的应用
- 跨进程服务设计
- SDK 封装思路
- App 与 Launcher 通信
- 命令回调与状态同步

## 高频面试题

- 为什么车机项目里 AIDL 很常见？
- 跨进程服务设计要关注哪些核心问题？
- SDK 封装如何屏蔽底层通信细节？

## 追问方向

- 接口版本演进、回调线程和连接重试。
- 命令分发、状态同步和幂等设计。
- 多端协同和系统服务边界。

## 易错点 / 对比点

- IPC 设计重点不只是“能调通”，还要考虑稳定性和兼容性。
- 回调和状态同步往往比单次命令调用更复杂。
- SDK 封装既要易用，也要保留错误处理能力。
