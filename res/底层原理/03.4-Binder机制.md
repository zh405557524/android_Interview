# 3.4 Binder 机制

## 本章目标

- 建立 Binder 作为 Android 核心 IPC 机制的完整认知。
- 能讲出 Binder 的角色模型、调用链路和为什么适合 Android。

## 核心知识点清单

- Binder 是什么
- Binder 驱动
- `ServiceManager`
- AIDL 原理
- Client / Server / Proxy / Stub
- Binder 线程池
- 一次 Binder 调用链路
- 为什么 Binder 适合 Android

## 高频面试题

- Binder 和传统 IPC 相比有什么优势？
- AIDL 背后的核心原理是什么？
- 一次 Binder 调用从客户端到服务端大致怎么走？

## 追问方向

- 驱动、内存映射和拷贝次数。
- `BpBinder`、`BBinder`、Stub、Proxy 的角色关系。
- Binder 线程池和调用阻塞风险。

## 易错点 / 对比点

- Binder 不只是“一个类”，而是驱动 + 协议 + 框架的一整套机制。
- AIDL 是 Binder 的使用方式之一，不等于 Binder 全部。
- 要区分 ServiceManager、AMS 等系统服务和普通业务 Service。
