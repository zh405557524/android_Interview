# 7.4 语音架构设计

## 本章目标

- 从工程视角理解语音系统的模块划分和扩展设计。
- 能回答语音引擎接入、多业务接入和 UI 协同等问题。

## 核心知识点清单

- 引擎层封装
- 业务层解耦
- 语义分发
- 命令分发
- UI 与语音流程协同
- 多业务接入的扩展性设计

## 高频面试题

- 语音架构为什么要做引擎层和业务层分离？
- 语义分发和命令分发分别解决什么问题？
- 多业务接入时如何避免语音系统越来越乱？

## 追问方向

- 状态机、路由和能力注册机制。
- UI 回调、对话态和异步结果协同。
- 扩展性、隔离性和故障兜底。

## 易错点 / 对比点

- 语音架构设计难点常在编排和状态管理，不只是 SDK 封装。
- 命令分发不等于简单 `if-else`。
- 多业务接入最怕强耦合和边界不清。
