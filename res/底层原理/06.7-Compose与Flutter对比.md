# 6.7 Compose 与 Flutter 对比

## 本章目标

- 从声明式 UI 视角对比 Compose 与 Flutter 的共性和差异。
- 能回答“为什么选它、如何混合落地、工程上怎么权衡”。

## 核心知识点清单

- 声明式 UI 思维
- 状态驱动 UI
- 重组机制对比
- 工程落地差异
- 混合开发思路

## 高频面试题

- Compose 和 Flutter 在声明式 UI 上有哪些共性？
- 二者状态管理和刷新机制有什么差异？
- 在 Android 团队里什么时候更适合选 Compose？

## 追问方向

- 宿主环境、生态成熟度和团队学习成本。
- 与原生 View 混用、与原生页面混合的难度。
- 工程落地中的构建、调试和性能差异。

## 易错点 / 对比点

- 两者都声明式，但运行时和宿主模型完全不同。
- Compose 深度绑定 Android 生态，Flutter 更跨平台。
- 比较时要区分技术能力和团队现实成本。
