# 1.3 Java 并发

## 本章目标

- 理解线程协作、同步机制与 JUC 核心组件之间的关系。
- 能把并发问题讲成“问题背景 + 可见性/有序性/原子性 + 解决方案”。

## 核心知识点清单

- 线程与进程
- `Runnable` / `Callable` / `Future`
- `synchronized`
- `volatile`
- `ThreadLocal`
- `wait` / `notify` / `notifyAll`
- `Lock` / `ReentrantLock`
- CAS
- AQS
- 线程池 `ThreadPoolExecutor`
- 并发容器
- 死锁、线程安全、可见性、有序性、原子性

## 高频面试题

- `synchronized` 和 `ReentrantLock` 有什么区别？
- `volatile` 为什么不能保证原子性？
- 线程池核心参数如何设置，拒绝策略怎么选？

## 追问方向

- happens-before、内存屏障和 JMM 基础。
- CAS 的 ABA 问题以及 `AtomicStampedReference` 思路。
- AQS 如何支撑锁、同步器和线程协作。

## 易错点 / 对比点

- `ThreadLocal` 解决的是线程隔离，不是共享同步。
- `wait/notify` 必须和对象监视器一起使用。
- 线程安全不只等于“加锁”，还包含无锁、不可变和线程封闭。
