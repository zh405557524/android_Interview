# 3.11 事件分发原理

## 本章目标

- 从系统输入链路视角理解触摸事件如何到达 View。
- 衔接 Input 子系统、窗口系统和应用 View 分发机制。

## 核心知识点清单

- `InputReader` / `InputDispatcher` 基础
- 事件到 View 的分发链路
- 焦点、窗口、View 关系

## 高频面试题

- 触摸事件从硬件输入到 View 消费大致怎么走？
- `InputDispatcher` 和 WMS 是什么关系？
- 为什么窗口焦点会影响事件分发？

## 追问方向

- 输入设备、事件读取和分发线程模型。
- `WindowInputEventReceiver` 到 `ViewRootImpl` 的链路。
- 焦点切换、手势导航和多窗口场景。

## 易错点 / 对比点

- 应用层事件分发只是系统输入链路的后半段。
- View 拿不到事件，很多时候是窗口层级或焦点问题。
- InputReader 和 InputDispatcher 分工不同，一个读入，一个路由。
