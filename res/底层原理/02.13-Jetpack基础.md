# 2.13 Jetpack 基础

## 本章目标

- 建立 Jetpack 常用组件的职责划分和协同关系。
- 能把现代 Android 架构组件和传统实现做对比。

## 核心知识点清单

- `Lifecycle`
- `LiveData`
- `ViewModel`
- Room
- Navigation
- `WorkManager`
- `DataBinding` / `ViewBinding`

## 高频面试题

- `ViewModel` 为什么能跨配置变更保存界面状态？
- `LiveData` 和 `Flow` 的差异是什么？
- `ViewBinding` 和 `DataBinding` 应该怎么选？

## 追问方向

- 生命周期感知组件的设计价值。
- Navigation 的回退栈和页面组织思路。
- `WorkManager` 与前台服务、定时任务的边界。

## 易错点 / 对比点

- `ViewModel` 不应该持有 Activity/Fragment 引用。
- `LiveData` 更偏 UI 感知，`Flow` 更偏通用异步流。
- `ViewBinding` 更轻量，`DataBinding` 更强调声明式绑定。
