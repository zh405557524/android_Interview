# 3.12 渲染机制

## 本章目标

- 理解 Android 从 UI 指令到屏幕显示的渲染主线。
- 能讲清软件绘制、硬件加速和 Surface 体系的关系。

## 核心知识点清单

- 软件绘制与硬件加速
- `RenderThread` 基础
- `Surface` / `SurfaceFlinger` 基础
- 双缓冲 / 三缓冲 基础认知

## 高频面试题

- Android 为什么需要硬件加速？
- `SurfaceFlinger` 在系统渲染中扮演什么角色？
- 双缓冲和三缓冲分别解决了什么问题？

## 追问方向

- UI 线程、RenderThread 和 GPU 的职责分工。
- BufferQueue、图层合成和显示时序。
- 动画卡顿、掉帧与渲染瓶颈分析。

## 易错点 / 对比点

- 硬件加速不代表所有绘制都没有 CPU 参与。
- `Surface` 是生产者和消费者之间的显示缓冲抽象。
- 三缓冲提升吞吐，但也可能增加延迟。
